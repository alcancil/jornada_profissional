## 🎯 Objetivo deste Capítulo

Até agora, focamos quase exclusivamente em competências técnicas – as chamadas **hard skills**. Porém, um currículo completo precisa mostrar também **como você se comporta no ambiente de trabalho**, resolve problemas e se relaciona com as pessoas.

Neste capítulo você aprenderá a:

- Diferenciar hard skills de soft skills de forma clara.
- Identificar quais soft skills são mais valorizadas para cada perfil de vaga (NOC, SOC, ISP, Enterprise, Automação, Cloud, etc.).
- Descrever soft skills no currículo **sem usar frases feitas** como "sou proativo" ou "aprendo rápido".
- Transformar experiências do cotidiano em evidências concretas de soft skills.

---

## 🧠 O que são Hard Skills?

**Hard skills** são habilidades técnicas, ensináveis e mensuráveis. São o "o quê" você sabe fazer.

| **Característica**                                | **Exemplo**                                                   |
|:---                                               |:---                                                           |
| São aprendidas em cursos, livros e certificações. | Configurar OSPF, escrever um script em Python.                |
| Podem ser testadas e comprovadas.                 | "Tenho CCNA válido", "Sei usar o comando show ip route".      |
| São específicas de uma área.                      | BGP é uma hard skill de redes; não serve para design gráfico. |

**No currículo:** hard skills aparecem na lista de **Competências Técnicas** e nas **Certificações**.

---

## 🧠 O que são Soft Skills?

**Soft skills** são habilidades comportamentais, emocionais e sociais. São o "como" você faz o que sabe.

| **Característica**                                    | **Exemplo**                                           |
|:---                                                   |:---                                                   |
| São desenvolvidas com experiência e autoconhecimento. | Saber ouvir um usuário insatisfeito e manter a calma. |
| São mais difíceis de mensurar.                        | Não há prova de "comunicação eficaz".                 |
| São transferíveis entre áreas.                        | Liderança serve para redes, marketing e gestão.       |

**No currículo:** soft skills aparecem indiretamente no **Resumo Profissional** e nas **Realizações**, quando você descreve como agiu.

---

## 📊 Comparação Direta: Hard vs Soft

| **Hard Skills**                       | **Soft Skills**                                         |
|:---                                   |:---                                                     |
| Configurar VLANs.                     | Explicar por que as VLANs são importantes.              |
| Escrever um script em Python.         | Documentar o script para que outros possam usá-lo.      |
| Corrigir um erro no Active Directory. | Manter a calma durante uma interrupção de serviço.      |
| Conhecer BGP.                         | Negociar com um provedor sobre a configuração do BGP.   |
| Criar um dashboard no Grafana.        | Apresentar o dashboard para a diretoria de forma clara. |

**A chave:** hard skills te dão o **meio**, soft skills te dão o **alcance**. Você pode ser o melhor técnico do mundo, mas se não sabe se comunicar, seu valor é limitado.

## 🗺️ Mapeamento de Soft Skills por Perfil de Vaga

Assim como fizemos com as competências técnicas, podemos mapear quais soft skills são mais valorizadas em cada perfil. Isso ajuda a escolher o que destacar no currículo e na entrevista.

### Tabela de soft skills mais valorizadas por perfil

| **Perfil**                          | **Soft skills mais importantes**                                                                                        |
|:---                                 |:---                                                                                                                     |
| **NOC** (Monitoramento e Operações) | Atenção a detalhes, capacidade de priorizar incidentes, comunicação clara em relatórios, resiliência sob pressão.       |
| **SOC** (Segurança)                 | Pensamento analítico, capacidade de investigação, curiosidade técnica, discrição e comunicação de riscos.               |
| **ISP** (Provedor)                  | Negociação com fornecedores, capacidade de planejamento de capacidade, comunicação com clientes, gestão de conflitos.   |
| **Enterprise** (Rede Corporativa)   | Visão sistêmica, capacidade de traduzir requisitos de negócio para soluções técnicas, trabalho em equipe, empatia.      |
| **Cloud** (AWS/Azure)               | Adaptabilidade (tecnologias mudam rápido), capacidade de aprender de forma autônoma, pensamento de custo (FinOps).      |
| **Automação / NetDevOps**           | Curiosidade, capacidade de documentar, pensamento lógico, paciência para depuração, cultura de colaboração (Git).       |
| **Infraestrutura** (Servidores)     | Planejamento de manutenção, gestão de tempo, comunicação com equipes de aplicação, proatividade na prevenção de falhas. |

## ✍️ Como descrever Soft Skills no currículo (sem parecer clichê)

O maior erro na hora de escrever soft skills é usar frases genéricas. O recrutador **não acredita** em "sou proativo" ou "aprendo rápido" – ele quer **evidências**.

### Exemplos de frases ruins (clichês)

| **Clichê**                | **Por que é ruim**                                                |
|:---                       |:---                                                               |
| "Sou proativo."           | Todo mundo diz isso. Não é provado.                               |
| "Tenho boa comunicação."  | Como você prova? Com números? Com exemplos?                       |
| "Trabalho bem em equipe." | Quem não diz isso? Mostre um caso onde a equipe dependeu de você. |
| "Sou resiliente."         | Isso é uma qualidade, mas sem contexto vira palavras vazias.      |

### Exemplos de frases boas (com evidência)

| **Soft skill**                | **Como descrever com impacto**                                                                                                            |
|:---                           |:---                                                                                                                                       |
| **Proatividade**              | "Identifiquei que os backups não estavam sendo testados e criei um script de validação automática, evitando perda de dados."              |
| **Comunicação**               | "Traduzi requisitos técnicos de rede para a equipe de negócios, alinhando as expectativas e reduzindo o retrabalho."                      |
| **Trabalho em equipe**        | "Colaborei com a equipe de segurança para implementar ACLs, garantindo que todos os requisitos fossem atendidos sem impacto na operação." |
| **Resiliência**               | "Mantive a calma e a comunicação durante um incidente crítico de rede, coordenando a equipe até a resolução total."                       |
| **Capacidade de aprendizado** | "Em 3 meses, migrei do suporte de TI para o time de redes, absorvendo conceitos de BGP e OSPF em laboratórios autodidatas."               |

**Regra de ouro:** Para cada soft skill, pergunte-se: **"Onde essa habilidade fez diferença?"** A resposta vira sua frase.

## ✅ Checklist de Validação de Soft Skills

Antes de colocar uma soft skill no currículo, verifique:

- [ ] A soft skill está **alinhada com o perfil da vaga**?
- [ ] Eu consigo **provar** essa soft skill com uma realização concreta?
- [ ] A frase de evidência usa um **verbo de ação** e descreve **um impacto real**?
- [ ] Eu evitei frases genéricas como "sou proativo" sem contexto?
- [ ] A soft skill que eu escolhi **realmente me define**? Não force uma que não seja verdadeira.

Se você respondeu **sim** para todas, você pode usar essa soft skill com confiança.
