## 🎯 Objetivo deste Capítulo

No Capítulo 9, você aprendeu o **conceito** de anonimização, as regras e o passo a passo manual. Agora, vamos colocar a teoria em prática com **exemplos reais** de currículos anonimizados para diferentes perfis profissionais.

Você verá:

- Três currículos completos (Analista de TI, Analista de Redes e Automação/NetDevOps).
- Cada um apresentado em sua versão **completa** (com dados pessoais) e **anonimizada**.
- Uma análise detalhada do que foi removido, substituído e mantido em cada caso.

O objetivo é fixar o processo e mostrar como a anonimização se aplica a diferentes contextos, sem depender de ferramentas automatizadas.

## 📊 Comparação Geral das Anonimizações

| **Perfil**        | **Dados removidos**                                          | **Dados mantidos**                                   | **Particularidade**                |
| :---              | :---                                                         | :---                                                 | :---                               |
| Analista de TI    | Nome, telefone, e-mail, LinkedIn, GitHub.                    | Cidade, empresa, cargo, habilidades, certificações.  | Nenhuma.                           |
| Analista de Redes | Nome, telefone, e-mail, LinkedIn, GitHub, links de projetos. | Cidade, descrição de projetos (sem links).           | Os links de projetos foram removidos para evitar rastreamento. |
| Automação         | Nome, telefone, e-mail, LinkedIn, GitHub, links de projetos. | Cidade, descrição de projetos (sem links).           | Projetos com links são removidos; as descrições ficam.                |

## 🧠 Lições Aprendidas com os Exemplos

- A anonimização é contextual: O que você remove depende do destino do currículo. Para um recrutador, você pode manter o LinkedIn; para uma LLM, você deve anonimizar tudo.
- Links são rastreáveis: Mesmo sem seu nome, um link pode identificar você (ex: github.com/anaribeiro). Sempre remova ou substitua por placeholders.
- A cidade/UF é segura e útil: Mantenha-a, pois ajuda o recrutador a saber sua localização.
- As descrições de projetos são seguras: Elas demonstram sua competência sem expor dados pessoais. Apenas remova os links.
- A anonimização não altera o conteúdo técnico: Seu currículo continua tendo o mesmo peso profissional, apenas perde a identificação pessoal.
