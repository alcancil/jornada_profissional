## 🎯 Objetivo deste Capítulo

O Capítulo 6 mostrou a diferença entre hard e soft skills. Agora, este capítulo ensina **como provar que você possui essas soft skills** – não com adjetivos, mas com **comportamentos observáveis**.

Você aprenderá a:

- Substituir frases genéricas por evidências concretas.
- Mapear cada soft skill a ações documentáveis.
- Utilizar seus próprios projetos (como laboratórios de rede) para demonstrar simultaneamente habilidades técnicas e comportamentais.
- Construir respostas sólidas para perguntas de entrevista baseadas em fatos reais.

---

## 🧠 O Problema: Soft Skills como Adjetivos Vazios

A maioria dos currículos lista soft skills assim:

- Proativo
- Boa comunicação
- Trabalho em equipe
- Liderança
- Organizado

Isso **não vale nada**. Qualquer pessoa pode escrever. O recrutador lê e pensa: *"Como sei que isso é verdade?"*

A verdade é que **uma soft skill não é um adjetivo – é um comportamento observado**. Se você não descreve o comportamento, a soft skill não existe no papel.

---

## 🔍 A Solução: Soft Skills Observáveis

Para demonstrar uma soft skill, você deve responder a três perguntas:

1. **Qual comportamento** demonstra essa habilidade?
2. **Em que situação** isso aconteceu?
3. **Qual foi o impacto** ou resultado?

A partir daí, você constrói uma frase de evidência.

## 📊 Tabela de Soft Skills × Comportamentos Observáveis

| **Soft Skill** | **Comportamentos observáveis** | **Exemplo de frase (currículo)** |
|:---             |:---|:---|
| **Comunicação** | Documentação, apresentações, treinamentos, tradução técnica para negócios. | *"Apresentei a nova arquitetura de rede para a diretoria, traduzindo conceitos técnicos em linguagem acessível."* |
| **Organização** | Documentação padronizada, inventário, controle de versões (Git), procedimentos claros. | *"Padronizei os backups de configuração utilizando Git, criando um histórico completo de alterações."* |
| **Liderança**   | Coordenação de projetos, distribuição de tarefas, mentoria de estagiários. | *"Coordenei a implantação de uma nova estrutura de VLANs, distribuindo atividades entre três equipes."* |
| **Trabalho em equipe** | Participação em projetos multidisciplinares, colaboração entre áreas. | *"Colaborei com o time de segurança para implementar ACLs, garantindo que todos os requisitos fossem atendidos."* |
| **Proatividade** | Antecipação de problemas, implementação de melhorias sem ser solicitado. | *"Identifiquei que os backups não estavam sendo testados e criei um script de validação automática."* |
| **Resiliência** | Manutenção da calma em incidentes críticos, capacidade de liderar a equipe durante pressão. | *"Mantive a comunicação e coordenação durante um incidente crítico de rede, até a resolução total."* |
| **Aprendizado contínuo** | Certificações, cursos, desenvolvimento de laboratórios próprios, autodidatismo. | *"Obtive CCNA e iniciei os estudos do CCNP, desenvolvendo laboratórios próprios e documentação técnica."* |
| **Pensamento analítico** | Troubleshooting, diagnóstico de problemas complexos, formulação de hipóteses. | *"Diagnostiquei um loop de STP que afetava 30 switches; isolei a causa e apliquei Root Guard."* |
| **Gestão do tempo** | Cumprimento de prazos, priorização de incidentes, entrega de projetos dentro do cronograma. | *"Cumpri rigorosamente os SLAs de atendimento, priorizando incidentes críticos e mantendo a equipe informada."* |
| **Adaptabilidade** | Migração para novas tecnologias, aprendizado de novas ferramentas em curto período. | *"Em 3 meses, migrei do suporte de TI para o time de redes, absorvendo conceitos de BGP e OSPF."* |

## ✍️ Como Transformar Comportamentos em Frases de Currículo

### Exemplo prático: Comunicação

| **❌ Ruim**              | **✅ Bom (comportamento)**                                                                    |
|:---                      |:---                                                                                            |
| "Tenho boa comunicação." | "Documentei a topologia de rede e criei um runbook de troubleshooting, utilizado pela equipe." |

### Exemplo prático: Organização

| **❌ Ruim**       | **✅ Bom (comportamento)**                                                       |
|:---               |:---                                                                               |
| "Sou organizado." | "Padronizei a nomenclatura de equipamentos e a documentação de endereçamento IP." |

### Exemplo prático: Liderança

| **❌ Ruim**                  | **✅ Bom (comportamento)**                                                                                       |
|:---                          |:---                                                                                                               |
| "Tenho perfil de liderança." | "Mentorei dois estagiários durante um projeto de migração, acompanhando cada passo e validando as configurações." |

**Regra de ouro:** Nunca escreva a soft skill. Escreva **o comportamento que prova que você a tem**.

## 🧪 Soft Skills em Laboratórios e Projetos Pessoais

Documentar laboratórios de rede, scripts Python e a jornada de estudos são **fontes riquíssimas** de evidências de soft skills. Cada laboratório bem feito demonstra várias habilidades simultaneamente.

### Exemplo: Laboratório de STP (do seu repositório)

| **Hard skill demonstrada**      | **Soft skill demonstrada**                                      |
|:---                             |:---                                                             |
| STP, RSTP, MSTP                 | **Organização:** Documentação com sumário, objetivos e imagens. |
| PVST+, EtherChannel, BPDU Guard | **Pensamento analítico:** Troubleshooting e análise de BPDUs.   |
| Python, Netmiko, tshark         | **Aprendizado contínuo:** Automação integrada ao lab.           |
| README com markdown claro       | **Comunicação:** Explicações didáticas e exemplos.              |
| Evolução STP → RSTP → MSTP      | **Proatividade:** Progressão planejada e documentada.           |
| Código comentado e scripts      | **Organização / Atenção aos detalhes.**                         |

**Ou seja:** o mesmo laboratório alimenta seu currículo com hard skills **e** soft skills, sem que você precise inventar nada.

## 🗺️ Conectando Soft Skills a Perguntas de Entrevista

Uma soft skill bem demonstrada no currículo pode ser facilmente expandida para uma resposta de entrevista. Basta seguir o fluxo:

> Soft skill → Comportamento → Situação → Ação → Resultado


| **Soft skill** | **Pergunta típica**                                                                  | **Resposta baseada em comportamento**                                |
|:---            |:---                                                                                  |:---                                                                  |
| Comunicação | "Conte uma situação em que precisou explicar algo técnico para uma pessoa não técnica." | "Quando apresentei a nova topologia de rede para a diretoria, usei analogias e evitei jargões, garantindo a aprovação do projeto." |
| Resolução de problemas | "Descreva um incidente crítico que você resolveu."                           | "Em um incidente de loop STP, isolei a causa, testei hipóteses e apliquei Root Guard, restaurando o tráfego." |
| Trabalho em equipe | "Como você lida com conflitos em equipe?" | "Em um projeto de migração de VLANs, alinhei expectativas entre times e criei um cronograma consensual." |

## ✅ Checklist de Validação de Soft Skills Observáveis

- [ ] A soft skill foi substituída por um **comportamento**? (não use adjetivos)
- [ ] A frase descreve uma **situação real**?
- [ ] A frase mostra **o que você fez** (ação) e **qual foi o resultado**?
- [ ] A evidência é **verificável** – pode ser defendida em entrevista?
- [ ] A frase não contém clichês como "sou proativo" ou "aprendo rápido"?

Se você respondeu **sim** a todas, sua soft skill está pronta.
